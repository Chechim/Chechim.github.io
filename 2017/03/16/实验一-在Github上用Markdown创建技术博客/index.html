实验一 在Github上用Markdown创建技术博客
一、实验目的

  1. 熟悉Github网站
  2. 掌握Markdown语法

二、实验任务

  1. 注册Github账号
  2. 创建Gihub Pages repository
  3. 发布第一篇博客
  4. 学习Markdown
  5. 利用Hexo生成博客并部署到Github

三、实验步骤与结果

  1. 注册Github账号
    帐号：Chechim
  2. 创建Gihub Pages repository
  3. 发布第一篇博客
  4. 学习Markdown
  5. 利用Hexo生成博客并部署到Github
    操作步骤：
    （1） 安装git和node
    （2） 设置npm镜像：npm config set registry https://registry.npm.taobao.org
    （3） 安装hexo：npm install -g hexo-cli
    （4） 初始化站点：hexo init myhexo 、 cd myhexo 、 npm install
    （5） 浏览本地站点：hexo server
    （6） 在Github上新建repository，名称：Chechim.github.io
    （7） 修改配置文件_config.yml中的相关信息
    （8） 安装hexo-deployer_git：npm install hexo-deployer-git –save
    （9） 配置git全局邮箱与名字
    （10）生成并部署博客：hexo generate 、 hexo deploy
    （11）生成新文章：hexo new “《我的第一个博客》”
    （12）用编辑器修改文章：myhexo\source_posts\《我的第一个博客》.md
    （13）下载主题pacman到本地E:\MD\hexo\themes
    （14）修改配置文件_config.yml中的相关信息
    （15）生成并发布到Github hexo d -g

四、实验小结
第一次做博客，做了很长时间，中间也经历了无数次的失败，但是通过学习和借鉴各种资料还是成功了。于我而言，在这次学习过程中不仅意识到了不能轻言放弃，同时也感受到了hexo的乐趣，希望通过以后的学习和探索能收获的更多。
